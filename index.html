The Problem

AI assistants are brilliant but forgetful.

Every conversation starts from zero. Context is lost. Knowledge is siloed. Your team re-explains the same things over and over.

LLMs forget everything

Every session resets. There's no persistent memory. Your AI can't build on what it learned yesterday.

RAG is clunky to maintain

Embedding pipelines, chunking strategies, stale indexes. Building memory on top of RAG alone is fragile and labor-intensive.

Your data, someone else's cloud

Third-party memory services mean your emails, tickets, and business context live on infrastructure you don't control.

How It Works

Three steps to persistent memory.

Locigram connects to your data, extracts structured facts, and makes them instantly recallable.

1

Ingest

Connect email, tickets, chat, and financial systems. Connectors auto-activate from environment variables. Push anything via webhook.

2

Extract

LLMs parse entities, relationships, and confidence from raw content. Noise is filtered. Facts become addressable memory units — locigrams.

3

Recall

Query via MCP or REST. High-confidence facts are promoted into truths by the decay-based scoring engine. Your AI remembers what matters.

Features

Built for developers who ship.

MCP-Native

First-class Model Context Protocol support. Any MCP-compatible AI assistant can read and write memory out of the box.

Own Your Data

Self-hosted on your infrastructure. Your emails, tickets, and business context never leave your network.

Truth Engine

Decay-based scoring promotes high-confidence, frequently-accessed facts into truths. Noise fades. Signal sharpens.

Connectors

Microsoft 365, Gmail, HaloPSA, NinjaOne, QuickBooks Online, webhooks. Add a connector by setting env vars — zero code.

Postgres + Qdrant

Structured storage in PostgreSQL. Semantic search in Qdrant. Battle-tested databases you already know how to operate.

Docker / K3s

One-command deployment with Docker Compose. Production-ready Kubernetes manifests for K3s clusters.

Quick Start

Running in under a minute.

Clone, configure, and start. Locigram runs anywhere Docker runs.

# Clone the repo
git clone https://github.com/locigram/locigram.git
cd locigram

# Configure
cp .env.example .env
# Set PALACE_ID, API_TOKEN, POSTGRES_PASSWORD
# Optional: LLM endpoints, connector env vars

# Start everything
docker compose -f deploy/docker/docker-compose.yml up

# API:  http://localhost:3000
# MCP:  http://localhost:3000/mcp

Architecture

Modular by design.

TypeScript monorepo. Each package owns its dependencies. Swap, extend, or contribute to any layer independently.

PackageDescription
@locigram/serverHono REST API + MCP server (Streamable HTTP). Docker entry point.
@locigram/pipelineIngest loop — raw content to LLM extraction, entity resolution, dedup, and storage.
@locigram/truthTruth engine — promotion logic, decay sweep, co-occurrence clustering, noise re-assessment.
@locigram/vectorQdrant wrapper. One collection per palace. Semantic search layer.
@locigram/connector-*Microsoft 365, Gmail, HaloPSA, NinjaOne, QuickBooks Online, Webhook. Plugin architecture — auto-registered from env vars.
