The Problem

AI assistants are brilliant but forgetful.

Every conversation starts from zero. Context is lost. Knowledge is siloed. Your team re-explains the same things over and over.

LLMs forget everything

Every session resets. There's no persistent memory across Claude.ai, ChatGPT, or OpenClaw. Your AI can't build on what it learned yesterday.

Context is siloed

What you tell Claude.ai doesn't carry to ChatGPT. What your ticketing system knows, your AI doesn't. Memory is fragmented across every tool you use.

Your data, someone else's cloud

Third-party memory services mean your emails, tickets, and business context live on infrastructure you don't control.

How It Works

Four steps to persistent memory.

Locigram connects to your data, extracts structured facts, builds a knowledge graph, and makes everything instantly recallable.

1

Ingest

Connectors pull from email, tickets, chat, and financial systems. Push anything via webhook. Auto-activate from env vars.

2

Extract

GLiNER NER pre-tags entities, then LLMs parse relationships and confidence. Noise is filtered before anything hits storage.

3

Graph

Memory nodes land in Postgres, Qdrant, and Memgraph simultaneously. Graph edges connect memories to agents, sessions, and loci.

4

Recall

Query via MCP or REST. High-confidence facts are promoted to truths. GraphRAG surfaces session-connected memories across time boundaries.

Features

Everything you need. Nothing you don't.

MCP-Native Core

First-class Model Context Protocol support. Claude.ai, ChatGPT, OpenClaw, and any MCP-compatible assistant can read and write memory out of the box. 12 MCP tools including recall, context, correction, and session recovery.

Graph Intelligence

Optional Memgraph layer mirrors every memory as a graph node connected to agents, sessions, and loci. memory_session_start runs GraphRAG traversal — surfaces old memories still connected to your active work, outside the normal time window.

Truth Engine

Decay-based scoring promotes high-confidence, frequently-accessed facts into truths. Based on the Ebbinghaus forgetting curve — memories fade at λ=0.6 by default. Noise evaporates. Signal sharpens over time.

NER Pre-Extraction (GLiNER)

Optional GLiNER server pre-tags entities (person, org, location, software, IP, date) before each LLM call. Results are injected as hints — improving extraction quality and reducing hallucinated entities. Fail-open: pipeline continues LLM-only if GLiNER is unreachable.

Connectors

Microsoft 365 (email + Teams), Gmail, HaloPSA, NinjaOne, QuickBooks Online, Obsidian vault, webhooks. Activate by setting env vars — zero code required. Pre-classified path skips LLM for structured financial and device data.

Own Your Data

Self-hosted on your infrastructure. Your emails, tickets, and business context never leave your network. Postgres is the single source of truth — Qdrant and Memgraph are derived and rebuildable from it.

Postgres + Qdrant + Memgraph

Structured storage in PostgreSQL. Semantic search via Qdrant. Graph traversal via Memgraph. Three storage layers, each with a purpose — all populated by durable background workers.

Session Monitor

Daemon watches OpenClaw agent JSONL session files. Every 5 messages it writes handoff summaries to Locigram, keeping live-handoff.md and active-context.json always current. Multi-agent fleets supported.

Docker / K3s

One-command deployment with Docker Compose. Production-ready Kubernetes manifests for K3s clusters with Longhorn PVCs, initContainer migrations, and namespace isolation per palace.

External LLM Clients

Claude.ai and ChatGPT connect natively.

Locigram runs a full OAuth 2.0 + PKCE authorization server. External LLM services authenticate as MCP clients and memories are automatically service-tagged and routed to the correct locus — no model configuration needed.

Claude.ai

Claude.ai uses dynamic client registration — it auto-registers itself on first connect. After connecting, tag it in one SQL line and every memory_remember call lands in sessions/claude automatically.

Claude.ai → OAuth PKCE flow → unique token
Bearer token → SHA256 lookup → client.service = "claude"
memory_remember() → sessions/claude

In Claude.ai: Settings → Feature Preview → Model Context Protocol → Add https://mcp.locigram.ai

ChatGPT

ChatGPT connectors require a pre-created OAuth client with a client ID and secret. Create one via the admin API and provide it in ChatGPT's connector settings.

ChatGPT → OAuth PKCE flow → unique token
Bearer token → SHA256 lookup → client.service = "chatgpt"
memory_remember() → sessions/chatgpt

Note: ChatGPT's redirect URI changes every time a new connector is created — update oauth_clients redirect_uris after each reconnect.

Supported service loci — each service gets its own isolated namespace:

sessions/claude sessions/chatgpt sessions/gemini sessions/perplexity sessions/copilot sessions/grok sessions/mistral sessions/llama sessions/other

OpenClaw Integration

Built for OpenClaw agents.

Locigram is the official memory backend for OpenClaw agents. Two layers of integration — MCP tools for recall, and session-monitor for continuous handoff.

MCP Server

Configure Locigram as an MCP server in OpenClaw via mcporter. Agents get 12 tools: memory_remember, memory_recall, memory_session_start, memory_correct, and more. Post-compaction recovery works out of the box — agents call memory_session_start and get back the last 7 days of decisions and context.

Session Monitor

A daemon that watches OpenClaw agent JSONL session files and generates handoff summaries. Every 5 messages it writes to Locigram, keeping live-handoff.md and active-context.json always current. Quick Start

Running in under a minute.

Clone, configure, and start. Locigram runs anywhere Docker runs.

# Clone the repo
git clone https://github.com/locigram/locigram.git
cd locigram

# Configure
cp .env.example .env
# Set PALACE_ID, API_TOKEN, POSTGRES_PASSWORD
# Optional: MEMGRAPH_URL, GLINER_URL, connector env vars

# Start
docker compose -f deploy/docker/docker-compose.yml up

# API:  http://localhost:3000
# MCP:  http://localhost:3000/mcp

Architecture

Modular by design.

TypeScript monorepo. Each package owns its dependencies. Swap, extend, or contribute to any layer independently.

Package / ComponentDescription
@locigram/serverHono REST API + MCP server (Streamable HTTP). OAuth 2.0 + PKCE authorization server for external LLM clients. Background workers: embed, graph, truth. Docker entry point.
@locigram/pipelineIngest loop — GLiNER NER pre-extraction, LLM entity + locus extraction, dedup by source_ref, entity resolution, storage. Pre-classified fast path for structured data (financial records, device inventory).
@locigram/truthTruth engine — Ebbinghaus decay scoring, tier promotion (hot/warm/cold), co-occurrence clustering, noise re-assessment. Nightly sweep CronJob.
@locigram/vectorQdrant wrapper. One collection per palace. Semantic search layer. Populated by embed-worker every 30s.
@locigram/dbDrizzle schema + raw SQL migrations. Runs as K8s initContainer on every deploy. Tables: palaces, locigrams, truths, entities, sources, retrieval_events, oauth_clients, oauth_access_tokens.
@locigram/connector-*Microsoft 365, Gmail, HaloPSA, NinjaOne, QuickBooks Online. Plugin architecture — auto-registered from env vars.
session-monitorOpenClaw session monitor daemon. Watches agent JSONL files, generates handoff summaries, supports multi-agent fleets with dynamic discovery and concurrent session tracking.
obsidian-auditoptionalWeekly LaunchAgent. LLM-evaluates each Obsidian vault note for indexing worthiness. Writes approved index to ~/.locigram/obsidian-index.json.
obsidian-syncoptionalNightly LaunchAgent. Summarizes approved Obsidian notes via LLM, upserts to Locigram with deep links. Locus: connectors/obsidian-sync.
MemgraphoptionalGraph layer. Graph-worker mirrors every locigram as a Memory node connected to Agent, Session, Palace, and Locus nodes. Powers GraphRAG in memory_session_start.
GLiNERoptionalNER pre-extraction server. Detects entities before each LLM call. Model: knowledgator/gliner-multitask-large-v0.5. Fail-open — pipeline continues LLM-only if unavailable.
